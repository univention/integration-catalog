"definitions" section of the Univention Integration Catalog.

The "definitions" listed here describe categories or capabilities the individual integrations can reference:

* "artifacts" describe how the integration is provided (for example only documentation or an installable package)
* "plaforms" describe on which deployment an integration is available
* "capabilities" describe which functionality is provided by an integration
* "products" describe with which Univention products the integration is compatible
* "offerings" describes in which way a user can get support or subscriptions
* "license" describes the license of the source code
* "technology" is a list of keywords for used standards or technologies

These definitions are meant to grow over time, so additional entries have to be expected. All definitions have a unique identifier which can be referenced in the integration descriptions. Entries in the definitions must not be removed even if they are not needed anymore, as the identifiers must not be re-used.
